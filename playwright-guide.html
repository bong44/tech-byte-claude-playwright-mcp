FRAMEWORK
Playwright MCP 기반 개발 프레임워크 주요기능
Claude Code + Playwright MCP로 구성한 TDD 기반 개발 프레임워크입니다.
화면 설계부터 TC 자동 생성·실시간 검증·회귀 테스트까지 하나의 사이클로 연결됩니다.
🎯
TDD 개발 사이클
data-testid 설계 → TC 자동 생성 → MCP 실시간 검증 → 회귀 테스트의 반복 사이클로 화면 개발과 테스트를 동시에 진행합니다.
🤖
AI 기반 TC 자동 생성
/gen-playwright-scenario 커맨드로 화면 소스의 data-testid를 분석해 표준 포맷의 TC 파일을 자동으로 생성합니다.
📊
결과 리포트 & 자동 문서화
TC별 성공/실패·실행 시간·스크린샷·트레이스가 HTML 리포트로 자동 저장됩니다. 별도 문서 없이 검증 이력이 누적됩니다.

기존 방식 vs Playwright MCP 기반 개발 프레임워크

❌ 기존 수동 개발·테스트
  • 화면 개발 후 직접 클릭하며 수동 확인
  • 테스트 시나리오 작성에 별도 시간 소요
  • 검증 결과를 문서로 남기기 번거로움
  • 회귀 테스트 시마다 반복 수동 실행
  • 버그 재현 어려움 (스크린샷 없음)
✅ Playwright MCP 기반 개발 프레임워크
  • data-testid 설계와 화면 개발을 동시에 진행
  • /gen-playwright-scenario로 TC 자동 생성
  • MCP로 selector·동작을 코드 실행 전 즉시 검증
  • npm test 한 번으로 전체 회귀 테스트 자동 실행
  • 스크린샷·트레이스·리포트 자동 저장으로 이력 관리
SETUP
환경 구성
Claude Code 프로젝트에 .claude 디렉터리 설정만 추가하면 됩니다.
1
Playwright 설치
기존 Vue 프로젝트에 Playwright를 추가합니다.
bash 터미널 
# Playwright 설치
npm install -D @playwright/test

# 브라우저 바이너리 설치 (Chromium)
npx playwright install chromium
2
.claude/mcp.json — Playwright MCP 등록
Claude Code가 브라우저를 직접 제어할 수 있도록 MCP 서버를 등록합니다. 이 파일이 있으면 프로젝트를 열 때 자동으로 활성화됩니다.
json .claude/mcp.json 
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"],
      "env": {
        "PLAYWRIGHT_BROWSERS_PATH": "0"
      }
    }
  }
}
💡
MCP 활성화 확인 Claude Code 터미널에서 /mcp를 입력하면 등록된 MCP 서버 목록과 상태를 확인할 수 있습니다. playwright 서버가 connected 상태여야 브라우저 조작이 가능합니다.
3
.claude/skills/ — 커스텀 스킬 등록
슬래시 커맨드로 호출할 수 있는 AI 스킬 파일을 등록합니다. gen-playwright-scenario.md를 배치하면 /gen-playwright-scenario 커맨드가 활성화됩니다.
디렉터리 구조 .claude/ 
.claude/
├── mcp.json                        # Playwright MCP 서버 등록
└── skills/
    └── gen-playwright-scenario.md  # 시나리오 자동 생성 스킬
📝
gen-playwright-scenario.md — 스킬 내용 요약 스킬 파일은 Claude Code가 커맨드 실행 시 따라야 할 절차를 마크다운으로 정의합니다. 이 스킬은 아래 순서로 동작합니다.
  1. 화면 정보 수집 & 소스 분석 — 화면 ID · 라우트 · 뷰 파일을 파악하고, grep으로 해당 파일의 data-testid를 전부 추출합니다.
  2. TC 파일 생성 — tests/{화면ID}.spec.js를 표준 템플릿으로 생성합니다. TC 번호는 TC-{화면ID}-001 체계를 따릅니다.
  3. 카테고리별 시나리오 포함 — 렌더링 · 조회 · 유효성 · CRUD · 예외 · 인증 총 6개 카테고리를 자동 작성합니다.
  4. helpers/auth.js 확인 — 로그인 헬퍼가 없으면 함께 생성합니다.
  5. 실행 가이드 안내 — 생성 완료 후 실행 명령어를 터미널에 출력합니다.

data-testid 작성 규칙

Playwright 선택자가 화면 구조 변경에 영향받지 않도록 모든 요소에 data-testid를 부여합니다.

🤖 AI 스킬 토큰 효율 증가
/gen-playwright-scenario 실행 시 Claude는 소스 전체를 읽는 대신 grep data-testid 한 번으로 화면 구조를 파악합니다. 일관된 네이밍 규칙이 있으면 추가 질의 없이 TC를 바로 생성하므로 입력 토큰이 줄고 생성 품질도 높아집니다.
🏗️ 화면 구조 변경에 독립적인 TC
CSS 클래스·XPath 기반 선택자와 달리 data-testid는 마크업 구조가 바뀌어도 TC를 수정할 필요가 없습니다. 리팩터링·디자인 변경 후에도 기존 TC를 그대로 회귀 테스트에 재사용할 수 있습니다.
🔍 실패 원인 즉시 특정
btn-save-010처럼 의미 있는 이름을 쓰면 TC 실패 로그만 봐도 어떤 컴포넌트에서 문제가 발생했는지 바로 파악됩니다. MCP 디버깅 시 Claude도 selector 의미를 문맥 없이 이해할 수 있습니다.
패턴 예시 설명
page-{화면ID}page-sscm0100화면 최상위 래퍼, beforeEach 대기에 사용
search-areasearch-area조회 조건 영역
btn-{동작}btn-search, btn-add-010동작 버튼
grid-{ID}-wrapgrid-csys010-wrap그리드 컨테이너
row-{그리드}-{idx}row-010-0그리드 행
cell-{그리드}-{컬럼}-{idx}cell-010-cmmnCd-0편집 가능한 셀 입력 필드
input-{필드}input-login-id폼 입력 필드
modal-confirmmodal-confirm확인 다이얼로그
vue 실제 적용 예시 — Sscm0100View.vue · LoginView.vue 
<!-- ① 화면 최상위 래퍼 -->
<div class="sscm0100" data-testid="page-sscm0100">

  <!-- ② 조회 조건 영역 -->
  <div class="search-area" data-testid="search-area">

    <!-- ③ 폼 입력 필드 -->
    <input v-model="cond.cmmnCdNm" type="text" data-testid="input-cmmn-cd-nm" />
    <select v-model="cond.useYn"         data-testid="sel-use-yn">...</select>

    <!-- ④ 동작 버튼 -->
    <button @click="handleSearch" data-testid="btn-search">조회</button>
  </div>

  <!-- ⑤ 그리드 컨테이너 -->
  <div class="grid-wrap" data-testid="grid-csys010-wrap">
    <button @click="handleAdd010"  data-testid="btn-add-010">추가</button>
    <button @click="handleSave010" data-testid="btn-save-010">저장</button>

    <!-- ⑥ 동적 행 · 셀 (v-for 인덱스 바인딩) -->
    <tr
      v-for="(row, idx) in codeList010"
      :data-testid=`row-010-${idx}`
    >
      <input type="checkbox"    :data-testid=`chk-010-${idx}` />
      <input v-model="row.cmmnCd" :data-testid=`cell-010-cmmnCd-${idx}` />
    </tr>
  </div>

<!-- ⑦ 로그인 폼 입력 필드 (LoginView.vue) -->
<input type="text"     data-testid="input-login-id" />
<input type="password" data-testid="input-login-pw" />
<button type="submit"  data-testid="btn-login">로그인</button>

WORKFLOW
Claude Code + Playwright 전체 프로세스
개발부터 검증까지, Claude Code와 Playwright MCP로 자동화된 워크플로우입니다.
1️⃣
화면 개발
data-testid 속성을 함께 작성하며 화면 구현
2️⃣
MCP로 즉시 검증
자연어로 브라우저 조작, selector 실시간 확인
3️⃣
시나리오 자동 생성
/gen-playwright-scenario로 TC 파일 자동 생성
4️⃣
전체 테스트 실행
npm test 로 전체 TC 실행 및 결과 확인
5️⃣
리포트 분석
HTML 리포트로 결과 확인, 실패 시 Trace 분석
6️⃣
반복 자동화
코드 변경 시마다 동일한 프로세스 반복 실행
IMPORTANT
시나리오 생성 이후엔 토큰 0원
Claude는 "만들 때"만 필요합니다. "실행할 때"는 완전히 독립적으로 동작합니다.
✅
npm test는 Claude와 무관합니다 한 번 생성된 .spec.js 파일은 순수한 JavaScript 코드입니다. 이후 npm test는 Node.js + Playwright 엔진이 단독으로 실행하며, Claude API 호출이 전혀 없습니다. 토큰 소비 없이 무한정 반복 실행이 가능합니다.
🤖 Claude가 필요한 순간
  • /gen-playwright-scenario — TC 최초 생성
  • MCP 브라우저 직접 조작·검증
  • 실패 원인 분석 요청
  • 기존 TC 수정·추가 요청
→ 화면당 1~2회 정도 발생하는 초기 투자
⚡ Claude 없이 무한 실행
  • npm test — 전체 회귀 테스트
  • npm run test:headed — 브라우저 확인
  • npm run test:ui — GUI 디버깅
  • CI/CD 파이프라인 자동 실행
→ 배포 전마다, PR마다, 매일 밤마다 비용 없이 실행
💡
한 번 투자, 무한 회수 Claude로 시나리오를 생성하는 데 쓰는 토큰은 일회성 비용입니다. 이후 코드 변경이 없다면 동일한 TC가 CI 파이프라인에서 하루에 몇 번이든 실행돼도 추가 비용은 0원입니다. 화면이 변경되거나 TC를 새로 추가할 때만 Claude를 다시 쓰면 됩니다.
