OVERVIEW
Playwright 핵심 기능
이 프로젝트에서 실제로 활용되는 Playwright의 기능들입니다.
🔄
자동 대기 (Auto-Wait)
클릭·입력 등 모든 동작에서 요소가 준비될 때까지 자동으로 대기합니다. 불필요한 sleep() 없이 안정적인 테스트를 작성할 수 있습니다.
📸
스크린샷 / 비디오 녹화
테스트 실패 시 자동으로 스크린샷과 실행 영상을 저장합니다. Claude Code가 이 결과를 읽어 즉시 원인을 파악합니다.
🧩
MCP 통합
Playwright MCP를 통해 Claude Code가 브라우저를 직접 조작하고 테스트 시나리오를 실시간으로 생성·검증할 수 있습니다.
📊
HTML 리포트
직관적인 HTML 리포트로 각 TC의 성공/실패 현황, 실행 시간, 스텝 상세 내역을 한눈에 확인합니다.
🤖
AI 시나리오 자동 생성
Claude Code의 /gen-playwright-scenario 커맨드로 화면 소스를 분석해 TC 파일을 자동으로 생성합니다.
⚡
빠른 실행 · 병렬 처리
Chromium 기반의 네이티브 브라우저 제어로 빠른 실행 속도를 제공하며, 여러 TC를 병렬로 실행할 수 있습니다.

기존 방식 vs Claude Code + Playwright MCP

❌ 기존 수동 개발·테스트
  • 화면 개발 후 직접 클릭하며 수동 확인
  • 테스트 시나리오 작성에 별도 시간 소요
  • 검증 결과를 문서로 남기기 번거로움
  • 회귀 테스트 시마다 반복 수동 실행
  • 버그 재현 어려움 (스크린샷 없음)
✅ Claude Code + Playwright MCP
  • 자연어로 브라우저 직접 조작·즉시 검증
  • /gen-playwright-scenario로 TC 자동 생성
  • 실행마다 스크린샷·리포트 자동 저장
  • npm test 한 번으로 전체 회귀 테스트
  • MCP로 selector 실시간 확인 및 디버깅

1
Playwright 설치
기존 Vue 프로젝트에 Playwright를 추가합니다.
bash 터미널 
# Playwright 설치
npm install -D @playwright/test

# 브라우저 바이너리 설치 (Chromium)
npx playwright install chromium
2
.claude/mcp.json — Playwright MCP 등록
Claude Code가 브라우저를 직접 제어할 수 있도록 MCP 서버를 등록합니다. 이 파일이 있으면 프로젝트를 열 때 자동으로 활성화됩니다.
json .claude/mcp.json 
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"],
      "env": {
        "PLAYWRIGHT_BROWSERS_PATH": "0"
      }
    }
  }
}
💡
MCP 활성화 확인 Claude Code 터미널에서 /mcp를 입력하면 등록된 MCP 서버 목록과 상태를 확인할 수 있습니다. playwright 서버가 connected 상태여야 브라우저 조작이 가능합니다.
3
.claude/skills/ — 커스텀 스킬 등록
슬래시 커맨드로 호출할 수 있는 AI 스킬 파일을 등록합니다. gen-playwright-scenario.md를 배치하면 /gen-playwright-scenario 커맨드가 활성화됩니다.

실행 방법 & 토큰 사용량
실행 방법에 따라 Claude Code 토큰 소비량이 크게 달라집니다. 목적에 맞는 방법을 선택하세요.
🚀
전체 테스트 실행
Claude 관여 최소
Claude는 명령어만 실행하고 결과 출력을 컨텍스트에 포함합니다. 결과 분석을 요청하지 않으면 토큰 소비가 매우 적습니다.
npm test
npx playwright test
👁️
Headed 모드 (브라우저 표시)
Claude 관여 최소
브라우저 창을 열어 테스트가 실행되는 모습을 시각으로 확인합니다. 시연이나 디버깅에 적합합니다.
npm run test:headed

# 느리게 실행 (시연용)
npx playwright test --headed --slow-mo 800
🎛️
UI 모드 (Playwright GUI)
Claude 관여 없음
Playwright가 제공하는 자체 GUI 앱을 별도로 열어 TC를 선택·실행합니다. Claude는 실행 명령어만 전달하므로 토큰을 거의 사용하지 않습니다.
npm run test:ui
🤖
MCP 직접 조작 + 검증
Claude 관여 높음
자연어로 Claude에게 지시하면 MCP를 통해 브라우저를 직접 조작하고 스크린샷을 반환합니다. 스크린샷 1장당 ~1,500–2,500 토큰이 소비됩니다.

네, 맞습니다 — npm test는 Claude와 무관합니다 한 번 생성된 .spec.js 파일은 순수한 JavaScript 코드입니다. 이후 npm test는 Node.js + Playwright 엔진이 단독으로 실행하며, Claude API 호출이 전혀 없습니다. 토큰 소비 없이 무한정 반복 실행이 가능합니다.
🤖 Claude가 필요한 순간
  • /gen-playwright-scenario — TC 최초 생성
  • MCP 브라우저 직접 조작·검증
  • 실패 원인 분석 요청
  • 기존 TC 수정·추가 요청
→ 화면당 1~2회 정도 발생하는 초기 투자
⚡ Claude 없이 무한 실행
  • npm test — 전체 회귀 테스트
  • npm run test:headed — 브라우저 확인
  • npm run test:ui — GUI 디버깅
  • CI/CD 파이프라인 자동 실행
→ 배포 전마다, PR마다, 매일 밤마다 비용 없이 실행
💡
한 번 투자, 무한 회수 Claude로 시나리오를 생성하는 데 쓰는 토큰은 일회성 비용입니다. 이후 코드 변경이 없다면 동일한 TC가 CI 파이프라인에서 하루에 몇 번이든 실행돼도 추가 비용은 0원입니다. 화면이 변경되거나 TC를 새로 추가할 때만 Claude를 다시 쓰면 됩니다.
